### 도메인 이름이란?

사람이 읽을 수 있는 웹 서버 주소로, 인터넷에서 사용.

도메인 이름의 필요성: IP 주소는 기억하기 어렵고 시간이 지남에 따라 변경될 수 있기 때문에 사람이 이해하기 쉬운 도메인 이름을 사용.

#### 도메인 이름 구조

도메인 이름은 마침표(.)로 구분된 여러 부분으로 구성되며 오른쪽에서 왼쪽으로 읽음. 

- 최상위 도메인 (TLD): 도메인 이름 뒤에 있는 서비스의 일반적인 목적을 나타냄. (.com, .org, .net 등)
- 라벨 (구성 요소): TLD 뒤에 오는 부분으로, 특정 하위 도메인을 나타냄. (예: www.example.com에서 'www'와 'example'은 라벨) 
- 2차 수준 도메인 (SLD): TLD 바로 앞에 있는 라벨을 의미. (예: www.example.com에서 'example'은 SLD)

#### 도메인 이름 구매

도메인 이름은 구매하는 것이 아니라, 일정 기간 동안 사용할 권리를 얻는 것.

- 등록 기관: 도메인 이름과 연결된 기술 및 관리 정보를 추적하는 회사 
- 사용 가능한 도메인 이름 찾기: 등록 기관 웹사이트의 "whois" 서비스를 이용하거나, 터미널에서 whois 명령어를 사용.

#### 도메인 이름 등록 절차
- 등록 기관 웹사이트 방문
- "도메인 이름 받기" 클릭
- 필요한 정보를 입력한 후 양식 제출
- 등록 기관에서 도메인 이름 등록 완료 시 알림

### DNS 작동 방식

#### DNS 새로 고침

등록 기관에서 도메인 정보를 생성하거나 업데이트하면 전 세계 모든 DNS 서버에 정보가 새로 고쳐져야 한다. 

#### DNS 요청 과정
- 브라우저 주소 표시줄에 도메인 이름 입력 (예: mozilla.org)
- 컴퓨터에서 해당 도메인 이름에 대한 IP 주소를 로컬 DNS 캐시에서 확인
- IP 주소가 캐시에 없으면 DNS 서버에 IP 주소 요청
- 컴퓨터가 IP 주소를 확인하면 브라우저가 웹 서버와 콘텐츠 협상
