## 인터넷

인터넷은 전 세계 수십억 명의 사람들이 정보에 접근하고, 친구 및 가족과 소통하고, 비즈니스를 수행하는 등의 목적으로 사용하는 현대 생활의 필수적인 부분입니다.

인터넷은 `상호연결된 컴퓨터의 글로벌 네트워크` 이며, 데이터를 교환하기 위해 표준 통신 프로토콜 세트를 사용합니다.

개발자는 인터넷이 무엇이며 어떻게 작동하는지, 몇 가지 기본 개념, 용어 및 인터넷에서 어플리케이션과 서비스를 구축하는데 사용되는 몇 가지 일반적인 프로토콜을 이해하는 것이 중요합니다.

### 인터넷 작동 방식

`인터넷은 장치와 컴퓨터 시스템을 표준화된 프로토콜 세트를 사용하여 서로 연결하여 작동합니다.` 

이러한 프로토콜은 장치 간에 정보가 교환되는 방식을 정의하고 데이터가 안정적이고 안전하게 전송되도록 합니다.

인터넷의 핵심은 상호 연결된 라우터의 글로벌 네트워크이며, 이 라우터는 다양한 장치와 시스템 간에 트래픽을 전달합니다. 

인터넷을 통해 데이터를 보냐면 데이터가 작은 패킷으로 분할되어 장치에서 라우터로 전송됩니다.

라우터는 패킷을 검사하고 다음 경로의 라우터로 전달합니다. 

이 프로세스는 패킷이 최종 목적지에 도달할 때까지 계속됩니다.

인터넷에서는 데이터를, 교환할 수 있도록 `다양한 기술과 프로토콜이 함께 작동` 합니다. 

DNS, HTTPS 및 SSL/TLS(Secure Sockets Layer/Transport Layer Security) 프로토콜이 대표적입니다.

### 인터넷 주소

인터넷에 연결된 모든 컴퓨터에는 `고유한 주소` 가 있어야 합니다.

이 주소를 IP 주소라고하며, nnn.nnn.nnn.nnn 형식입니다(여기서 nnn은 0~255 사이의 숫자).


### 프로토콜 스택 및 패킷

컴퓨터가 인터넷 상의 다른 컴퓨터와 대화하는 방식은 `프로토콜 스택` 을 통해 이루어집니다.

인터넷에서 사용되는 프로토콜 스택은 사용되어지는 두가지의 주 통신 프로토콜로 인해 TCP/IP 프로토콜 스택이라고 합니다.

인터넷을 통해 전송되는 데이터는 `패킷` 이라는 관리 가능한 청크(덩어리)로 전송됩니다.

### 네트워킹 인프라

인터넷 서비스 제공업체(ISP)는 전화 접속 고객을 위한 모뎀 풀을 관리, 유지합니다. 

패킷은 전화 네트워크와 ISP의 로컬 장비를 통과한 후 ISP의 백본이나 ISP가 구매한 대역폭의 백본으로 라우팅 됩니다.

그 다음 패킷은 일반적으로 여러 라우터를 거쳐 여러 백본, 전용 회선 및 기타 네트워크를 통해 대상 컴퓨터에 도달할 때까지 이동합니다.

### 인터넷 인프라

인터넷 백본은 서로 연결된 여러 개의 대규모 네트워크로 구성됩니다.

이러한 대규모 네트워크를 `네트워크 서비스 제공업체` 또는 `NSP` 라고 합니다.

### 인터넷 라우팅 계층 구조

패킷은 인터넷을 통해 어떻게 전달될까요? 

인터넷에 연결된 모든 컴퓨터가 다른 컴퓨터의 위치를 알고 있을까요?

패킷이 인터넷 상의 모든 컴퓨터에 '브로드캐스트' 될까요?

질문에 대한 답은 모두 '아니오' 입니다.

패킷을 대상으로 전달하는데 사용되는 정보는 인터넷에 연결된 각 라우터가 보관하는 라우팅 테이블에 포함되어 있습니다.

라우터는 패킷 스위치 입니다.

### 도메인 이름 및 주소 확인

`도메인 이름 서비스` 또는 `DNS` 는 인터넷 상의 컴퓨터 이름과 해당 IP주소를 추적하는 분산 데이터베이스입니다.

### 인터넷 프로토콜 다시 살펴보기

인터넷이 제대로 작동하려면 여러 통신 프로토콜이 필요합니다. 여기에는 TCP 및 IP 프로토콜, 라우팅 프로토콜, 매체 액세스 제어 프로토콜, 애플리케이션 수준 프로토콜 등이 포함됩니다.

#### 애플리케이션 프로토콜 : HTTP 및 월드 와이드 웹

인터넷에서 가장 일반적으로 사용되는 서비스 중 하나는 월드와이드 웹(WWW)입니다.

웹을 작동시키는 애플리케이션 프로토콜은 `하이퍼텍스트 전송 프로토콜` 또는 `HTTP` 입니다.

HTTP는 웹 브라우저와 웹 서버가 인터넷을 통해 서로 통시하는데 사용하는 프로토콜입니다.

HTTP는 TCP계층 위에 위치하고 특정 애플리케이션이 서로 통신하는데 사용되기 때문에 애플리케이션 수준 프로토콜입니다.

#### 애플리케이션 프로토콜 : SMTP 및 이메일

일반적으로 사용되는 또 다른 인터넷 서비스로는 이메일이 있습니다.

이메일은 SMTP(Simple Mail Transfer Protocol)라는 애플리케이션 수준 프로토콜을 사용합니다.

#### 전송 제어 프로토콜(TCP)

프로토콜 스택의 애플리케이션 계층 아래에는 TCP 계층이 있습니다.

`TCP는 애플리케이션 프로토콜을 대상 컴퓨터의 올바른 애플리케이션으로 라우팅합니다. `

이를 위해 포트 번호가 사용됩니다.

#### 인터넷 프로토콜(IP)
TCP와 달리 `IP는 신뢰할 수 없는 비연결형 프로토콜` 입니다. 

IP의 역할은 다른 컴퓨터로 패킷을 보내고 라우팅 하는 것입니다.
